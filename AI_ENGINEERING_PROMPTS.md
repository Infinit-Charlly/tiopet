# AI_ENGINEERING_PROMPTS.md

TioPet — AI Engineering Prompt Library

Author: Charlly Acuña  
Purpose: Provide a professional set of prompts for AI-assisted development of the TioPet ecosystem.

This document enables AI tools to quickly understand how to contribute effectively to the project.

---

# Philosophy

TioPet is built with the belief that technology should improve life.

This platform is created:

• for pets who cannot speak
• for families who trust caregivers
• for communities that care for animals

Artificial intelligence is used here as a tool to help humans build systems that protect and care for animals.

The human remains the architect.

AI becomes the assistant.

---

# 1. Repository Analysis Prompt

Use when starting work with any AI.

Prompt:

Analyze this repository and the file PROJECT_MASTER_CONTEXT.md.

Explain:

1. The system architecture
2. The core features of the platform
3. The technology stack
4. The next logical development steps

Do not rewrite the project. Only analyze and propose improvements aligned with the existing architecture.

---

# 2. Feature Development Prompt

Use when implementing new features.

Prompt:

Analyze the repository and propose a clean implementation for the following feature.

Requirements:

• maintain current architecture
• use TypeScript best practices
• avoid unnecessary dependencies
• ensure scalability

Provide:

1. architecture explanation
2. required files
3. implementation code
4. potential edge cases

---

# 3. Bug Investigation Prompt

Prompt:

Analyze the repository and identify the cause of the following bug.

Explain:

• root cause
• affected components
• safest fix

Then provide the corrected implementation.

Avoid rewriting unrelated parts of the system.
