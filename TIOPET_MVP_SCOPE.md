# TIOPET_MVP_SCOPE.md

TioPet — MVP Scope Definition

Author: Charlly Acuña  
Purpose: Define what features belong to the Minimum Viable Product (MVP) and what should be postponed.

---

# 1. MVP Philosophy

The MVP must deliver a **complete but minimal experience** that allows TioPet to operate in a real environment.

Goals of the MVP:

- Validate the pet care service model
- Provide operational tools for staff
- Create trust with pet owners
- Keep development manageable

The MVP must **avoid unnecessary complexity**.

---

# 2. Core MVP Modules

The MVP will include the following systems.

---

# 3. User System

Users must be able to:

- access the application
- create a profile
- manage their pets

Features included:

- anonymous login (initial)
- optional Google login
- user profile
- pet profiles

---

# 4. Pet Profiles

Users must be able to register their pets.

Fields:

- name
- species
- breed
- age
- weight
- notes
- photo

Each pet becomes selectable when creating bookings.

---

# 5. Booking System

This is the **core feature of the MVP**.

Users must be able to:

- select pet
- choose service plan
- select dates
- choose add-ons
- confirm reservation

Booking statuses:

- pending
- confirmed
- cancelled
- completed

---

# 6. Booking History

Users must be able to see:

- past bookings
- upcoming bookings
- booking status
- booking details

The history screen must clearly show the lifecycle of reservations.

---

# 7. QR Check-in System

Each reservation should generate a QR code.

QR uses:

- pet drop-off verification
- pet pickup verification

Benefits:

- security
- correct owner identification
- operational clarity

---

# 8. Caregiver App (Basic Version)

Staff must be able to:

- view assigned pets
- perform check-in
- perform check-out
- log care events
- upload photos

This generates the **care timeline** visible to pet owners.

---

# 9. Care Timeline

Owners should be able to see the pet's activity timeline.

Possible events:

- check-in
- feeding
- snack
- playtime
- walk
- nap
- grooming
- photo update
- check-out

This is a major **trust feature**.

---

# 10. Add-ons

Basic add-ons may include:

- snack
- grooming
- bath
- walk

Add-ons should be configurable in the future.

---

# 11. Store Integration (Simple Version)

The MVP may include a **simple product catalog**.

Products may include:

- food
- snacks
- toys
- hygiene items

Customers may add products to their booking.

Complex inventory systems are **not required in MVP**.

---

# 12. Notifications (Basic)

Basic notifications should include:

- booking confirmed
- pet checked in
- photo uploaded
- pet ready for pickup

Delivery channels:

- push notifications

---

# 13. Admin Panel (Basic)

Admin tools must allow:

- viewing bookings
- viewing pets
- viewing owners
- viewing caregivers
- viewing timeline events

Advanced analytics are not required yet.

---

# 14. Features NOT Included in MVP

The following ideas are **intentionally postponed**.

Not included yet:

- AI behavior monitoring
- bark detection
- camera analytics
- external caregiver marketplace
- advanced payment systems
- franchise management tools
- complex inventory systems
- advanced analytics dashboards

These belong to **future phases**.

---

# 15. MVP Success Criteria

The MVP is successful if:

- users can register pets
- users can create bookings
- staff can manage pets during their stay
- owners can see pet care updates
- pets are safely checked in and out

If these goals are achieved, the platform is operational.

---

# 16. Guiding Rule

If a feature does not directly help:

- booking pets
- caring for pets
- updating owners

then it probably **does not belong in the MVP**.
